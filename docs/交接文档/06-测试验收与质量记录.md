## 8. App Store 前测试建议

上架前建议至少覆盖：

| 类型 | 样例 |
| --- | --- |
| 闲聊 | 你好 |
| 咨询 | 押金不退一定有效吗 |
| 生成完整 | 生成租房合同，提供双方、地址、金额、租期 |
| 生成缺失 | 只说“帮我生成租房合同” |
| 审查红色风险 | 随意入户、随意扣押金 |
| 审查黄色风险 | 维修责任不清、费用承担不清 |
| 长文本 | 5000 字合同 |
| 文件 | PDF、DOCX、TXT |
| 弱网 | 30 秒以上等待 |
| 失败 | 后端不可用 |

## 9. 验收结论

当前前端层面已经可以交接。后续验收重点转向正式后端、正式 Dify 工作流、iOS 真机和 App Store 审核材料。
