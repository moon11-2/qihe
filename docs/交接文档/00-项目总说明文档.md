## 2. 项目一句话说明

「契合」是一个面向普通用户的移动端 AI 合同助手。用户可以在首页直接输入自然语言，系统自动判断是合同审查、合同生成还是普通咨询；也可以从功能页手动进入合同审查或合同生成工作台。

当前产品只保留两个核心功能：

- 合同审查
- 合同生成

首页的咨询/闲聊只作为引导和兜底，不作为第三个核心业务模块。

### 3.2 Dify/AI 接入验证

已完成统一服务层：

```text
src/services/dify.ts
```

服务层支持：

- Mock 模式
- Dify 模式
- Dify 返回 JSON 解析
- Dify 非标准返回的兼容兜底
- `consult` 咨询分支
- `review` 审查分支
- `generate` 生成分支

当前本地 H5 可通过本机代理请求真实 Dify：

```text
http://127.0.0.1:8787/api/dify
```

本机 Dify 代理脚本：

```text
scripts/dify-proxy.mjs
```

已验证：

- 首页闲聊会真实请求 Dify，返回后留在聊天流。
- 首页合同生成会进入生成进度浮层，返回后进入生成结果页。
- 首页合同审查会进入审查进度浮层，返回后进入审查结果页。
- 功能页合同审查和合同生成可以调用同一服务层。

## 5. 如果要上架 App Store，后续必须做什么

### 5.1 工程和技术准备

1. 确认 iOS wrapper 使用正式产品名、图标、Bundle ID。
2. 同步最新 H5 到 iOS 工程。
3. 配置 Apple Developer Team。
4. 接入正式 HTTPS 后端，不在 App 包里保存 Dify Key。
5. 真机测试文件上传、合同生成、合同审查、结果复制和导出。
6. 做弱网、超时、失败重试和错误提示。
7. 检查 WKWebView 下滚动、键盘遮挡、底部安全区、文件选择。

### 5.2 后端和 Dify 准备

1. 搭建正式后端代理。
2. 服务端保存 Dify API Key。
3. 对前端提供稳定接口。
4. Dify 工作流按本文档的 JSON 输出契约发布。
5. 训练数据和规则文档整理成知识库或规则提示。
6. 建立测试样例，确保审查/生成不是模板套答。

### 5.3 合规和审核准备

1. 明确产品是 AI 辅助工具，不提供律师法律意见。
2. 在 App 内和隐私政策中说明用户上传合同文本的处理方式。
3. 不承诺自动生成结果具备法律效力。
4. 提供用户协议、隐私政策、免责声明。
5. 提供支持邮箱或支持网页。
6. App Store 隐私信息如实填写。

### 5.4 发布材料准备

1. App 图标。
2. App Store 截图。
3. App 简介。
4. 关键词。
5. 演示视频或发布会视频。
6. PPT 和产品说明。
