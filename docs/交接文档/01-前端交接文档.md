# 前端交接文档

日期：2026-07-04  
项目名称：契合  
技术栈：uni-app Vue3 + TypeScript H5  
目标端：H5、iOS WKWebView wrapper

## 1. 文档用途

本文档交接当前前端已经实现的页面、交互、状态、关键文件和验证方式。后续前端同学如果继续维护，应先阅读本文档。

## 2. 前端职责边界

当前前端负责：

- 用户输入合同内容或生成需求。
- 首页自由输入和意图分流。
- 合同审查和合同生成工作台。
- 调用统一服务层。
- 展示 Dify/Mock 返回的结果。
- 本地历史记录。
- 设置页本地偏好。
- 移动端视觉和交互。

当前前端不负责：

- 生产环境保存 Dify API Key。
- 生产后端代理。
- 云历史记录。
- 账号登录。
- 付费和用量统计。
- 法律结论的最终有效性。

```text

```

## 5. 首页交互逻辑

首页支持三种结果：

1. 判断为合同审查：显示审查进度浮层，Dify 返回完整结果后进入审查结果页。
2. 判断为合同生成：显示生成进度浮层，Dify 返回完整结果后进入生成结果页。
3. 判断为咨询/闲聊：留在聊天流，显示动态等待气泡，Dify 返回后更新聊天回复。

当前首页已经修复：

- 在线模式下不再先走本地关键词模板，而是先请求 Dify。
- 闲聊显示“正在理解你的问题”动态气泡。
- 审查/生成显示功能页同款处理浮层。
- Dify 返回完整结果后不再重复二次执行。

## 9. 本地历史

当前历史保存方式：

```text
uni.setStorageSync("qihe-history", history)
```

特点：

- 只保存在本机。
- 不上传云端。
- 支持搜索。
- 支持全部/审查/生成筛选。
- 支持单条删除。
- 设置页支持清空历史。

后续如接云历史，应新增后端接口，不要破坏当前本地历史兜底。

## 10. 设置页

前端不会显示任何 API Key。

## 11. 服务层

统一服务层：

```text
src/services/dify.ts
```

主要导出：

- `runContractAnalysis(payload)`
- `isDifyEnabled()`
- `getServiceModeLabel()`

前端根据环境变量判断模式：

```text
VITE_DIFY_ENABLED=true/false
VITE_DIFY_PROXY_PATH=/api/dify 或 http://127.0.0.1:8787/api/dify
```

注意：

- 生产环境不应让前端直接访问 Dify 官方接口。
- 生产环境应改为访问自有 HTTPS 后端。
- Dify Key 只能在服务端。

```bash

```

```bash
npm run dev:h5 -- --host 127.0.0.1
```

访问：

```text
http://127.0.0.1:5173/
```

类型检查：

```bash
npm run type-check
```

构建：

```bash
npm run build:h5
```

## 13. 已知限制

- 当前主界面集中在一个大文件 `index.vue`，后续可拆组件，但交接前为降低风险未做大拆分。
- 当前历史为本地历史。
- 当前文件上传在 H5/开发环境可验证，生产和 iOS 真机需后端代理配合。
- 当前 Dify 工作流仍需后续同学继续优化规则和品牌文案。
- 当前 iOS wrapper 未自动同步本轮最新 H5，需要手动跑同步脚本。
