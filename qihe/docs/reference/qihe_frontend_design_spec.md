# 契合前端设计定稿 v3

## 结论

前端设计以后按 `qihe_redesign_v3.html` 的方向执行：`文书 · 印鉴 × 法务藏青`。

这版比蓝白 AI 工具风更适合合同产品。它把“合同、文书、公章、审查报告”的视觉线索放到界面里，同时保留第一版产品边界：自由聊天、合同审查、合同生成、本地历史、PDF / Word / TXT 上传。

## 产品边界

必须保留：

- 自由聊天：走 `/api/chat`，调用千问。
- 合同审查：调用千问。
- 合同生成：第一版调用千问，后续预留 Dify provider。
- 文件上传：只支持 `PDF / Word / TXT`。
- 历史记录：只在前端本地保存。

必须删除或隐藏：

- 登录、会员、客服。
- 底部导航。
- 法律研究、类案、法条检索、案例检索、工商检索。
- 图片上传、图片 OCR。

说明：v3 结果卡里的 `法条依据` 可以作为合同审查报告内容展示，但不能做成独立的法条检索入口。

## 视觉基准

主题：`文书 · 印鉴 × 法务藏青`

颜色：

- 墨色：`#1D2129`，正文和主要图标。
- 法务藏青：`#23405F`，主按钮、发送键、Tab 高亮、流程节点、依据说明。
- 朱砂：`#C23528`，品牌印章、高风险、待补充字段，不用于普通按钮。
- 纸色：`#F7F5F0`，主背景。
- 卡片纸色：`#FEFDFB`，表单、卡片和文书预览。
- 线色：`#E5E1D8`，普通分割线。
- 强线色：`#D0CBBE`，输入框、虚线框、手机边界内分割。
- 赭黄：`#B07F1E`，中风险。
- 松绿：`#3D7A5C`，完成状态。

字体：

- 标题、文书、印章：`Songti SC`, `Noto Serif SC`, `STSong`, `SimSun`, serif。
- UI 正文：`-apple-system`, `BlinkMacSystemFont`, `PingFang SC`, `Microsoft YaHei`, sans-serif。

图标：

- 统一使用 SF Symbols 风格线性图标。
- 描边约 `1.6px - 1.8px`。
- 不使用花哨彩色功能图标。

圆角：

- 大卡片：`18px`。
- 页面卡片和表单：`12px`。
- 小控件：`7px - 10px`。
- 手机框原型：`54px`。

## 品牌元素

品牌标识采用朱砂印章：

- 首页大印章显示 `契`。
- 顶栏小印章显示 `契`。
- 印章边角使用 26% 圆角，模拟方章但不做过度古风。
- 朱砂只服务品牌和风险，不承担主操作。

品牌文字：

- 主标题：`契合`。
- 副标题：`AI 合同审查与生成助手`。

## 页面结构

### 1. 首页

保留元素：

- 左上菜单按钮：打开历史抽屉。
- 右上新建对话按钮。
- 中间朱砂印章 + `契合`。
- 副标题：`AI 合同审查与生成助手`。
- 大输入框：`输入合同内容，或描述你想生成的合同……`。
- 输入框按钮：附件、发送。
- 两个核心入口：`合同审查`、`合同生成`。
- 信任说明：`逐条审查 · 标注法条依据 · 历史仅本地保存`。
- 最近记录：默认 3 条，右侧 `展开`。

不出现：

- 底部导航。
- 法律研究、类案、法条、案例、客服、会员。
- 图片上传入口。

### 2. 历史抽屉

定位：本地历史，不是账号中心。

内容：

- 顶部品牌：小印章 + `契合`。
- 状态说明：`本地保存 · 不上传`。
- 标题：`历史对话`。
- 搜索图标。
- 历史列表，使用 `壹 / 貳 / 叁...` 编号。
- 底部：`已显示全部内容`、`清空历史`。

交互：

- 点击记录恢复本地会话。
- 删除、清空只操作 localStorage。

### 3. 聊天 / 过程页

状态：

- 普通聊天：展示气泡。
- 识别为生成：展示进入生成的卡片。
- 识别为审查：展示进入审查的卡片。
- 不确定：展示 `合同审查 / 合同生成` 二选一。
- 处理中：展示流程节点。

审查流程节点：

1. 解析文本
2. 识别主体
3. 条款比对
4. 出具报告

底部输入框：

- 占位：`请输入问题`。
- 只保留附件和发送。

### 4. 合同审查页

默认极简：

- 标题：`合同审查`，`审查` 用藏青强调。
- 大输入区：`粘贴合同文本`，带文书横线背景。
- 上传入口：`上传 PDF / Word / TXT`。
- 文件大小提示：`20MB 以内`。
- 折叠项：`更多信息（可选）`。
- 主按钮：`开始审查`。
- 免责声明：`AI 辅助审查，不构成法律意见`。

折叠项字段：

- 合同类型。
- 我的立场。
- 关注重点。

### 5. 合同生成页

默认极简：

- 标题：`合同生成`。
- 大输入区：描述生成需求。
- 折叠项：`补充要求（可选）`。
- 主按钮：`生成合同`。
- 免责声明：`AI 辅助起草，不构成法律意见`。

折叠项字段：

- 合同类型。
- 我的身份。
- 特殊约定。

### 6. 审查结果页

顶部 Tab：

- `原文`
- `风险`
- `主体`

风险页内容：

- 报告标题：`合同审查报告`。
- 摘要：风险概述。
- 依据说明：`审查依据：中国大陆现行法律`。
- 等级印章：例如 `C / 中等风险`。
- 统计条：风险数量、综合评分、风险等级。
- 风险卡列表。

风险卡必须包含：

- 风险等级印章：高风险用朱砂，中风险用赭黄。
- 风险标题。
- 涉及条款，例如 `涉及条款 · 第四条 付款方式 / 第六条 验收`。
- 风险分析。
- 修订建议。
- 建议替换条款。
- 法条依据行。

注意：`法条依据` 只是结果字段展示，不是检索功能。

### 7. 主体页

正常状态：

- 展示甲方、乙方、金额、期限、合同类型、司法辖区等。

空状态：

- 不臆造字段。
- 展示空印章和提示：`未识别到乙方、金额或期限等信息，请确认合同文本是否完整。`

### 8. 生成结果页

内容：

- 顶部：小印章 + `合同草案`。
- 文书预览：使用合同纸张卡片。
- 文书注记：`草案 · 契合起草 · 参照《民法典》合同编体例`。
- 待补充字段用朱砂虚线标签。
- 签署前清单。
- AI 免责声明。

按钮：

- `复制全文`
- `导出`
- `继续修改`

## 接口映射

### `/api/chat`

按返回 `type` 分流：

- `chat`：聊天气泡展示 `reply`。
- `route`：根据 `intent` 进入审查或生成。
- `need_input`：展示二选一卡片。
- `error`：展示错误。

### `/api/files/upload`

前端文件选择器只允许：

- `.pdf`
- `.doc`
- `.docx`
- `.txt`

不允许图片、压缩包、音视频。

### `/api/contracts/run`

审查：

- `mode=review`。
- 渲染 `review_result.result`。

生成：

- `mode=generate`。
- 渲染 `generate_result.result`。

## 本地保存

推荐 localStorage key：

```txt
qihe.local.records
```

结构：

```json
{
  "id": "local_record_123",
  "type": "review",
  "title": "房屋租赁合同审查",
  "created_at": "2026-07-03T18:30:00+08:00",
  "status": "complete",
  "conversation_id": "conv_xxx",
  "messages": [],
  "result": {}
}
```

首页展示最新 3 条；历史抽屉展示全部。

## 前端组件建议

- `QiheSeal`：朱砂印章。
- `PhoneShell`：移动端页面壳。
- `TopBar`：顶部栏。
- `HomeComposer`：首页输入框。
- `ContractEntryCard`：合同审查 / 合同生成入口。
- `HistoryDrawer`：本地历史抽屉。
- `ChatBubble`：聊天气泡。
- `RouteChoiceCard`：审查 / 生成二选一。
- `ReviewProgressSteps`：解析文本、识别主体、条款比对、出具报告。
- `ContractReviewForm`：审查表单。
- `ContractGenerateForm`：生成表单。
- `ResultTabs`：原文 / 风险 / 主体。
- `RiskCard`：风险卡。
- `LawReference`：法条依据展示行。
- `SubjectFacts`：主体识别结果。
- `GeneratedContractSheet`：合同草案纸张预览。
- `MissingFieldChips`：待补充字段。

## 后端字段建议

为了支持 v3 结果页，审查结果可以在原有字段上补充这些展示字段：

```json
{
  "review_basis": "中国大陆现行法律",
  "clause_reviews": [
    {
      "clause_ref": "第四条 付款方式 / 第六条 验收",
      "legal_basis": ["《民法典》第五百零九条", "《民法典》第六百二十一条"]
    }
  ]
}
```

这些字段缺失时，前端隐藏对应行，不要写死占位法条。

## 验收标准

- 首页只出现合同审查、合同生成两个核心入口。
- 上传入口只出现 `PDF / Word / TXT`。
- 普通聊天不会触发合同审查或合同生成任务。
- 历史刷新后仍在本地。
- 大模型或 Dify 原始 JSON 不直接暴露给用户。
- 结果页可以展示法条依据，但没有独立法条检索入口。
- 所有主操作使用法务藏青，风险和印章使用朱砂。
- 生成结果页保留 `复制全文`、`导出`、`继续修改`。
